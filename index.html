The Problem

Files have been
silent for 50 years.

You open a file. You read it. You close it. The file learned nothing. It remembers nothing. It can answer nothing. That ends now.

📄

A PDF is just pixels. A DOCX is just XML. A JPG is just bytes. They carry data — but no understanding of that data.

Architecture

Four layers.
One file.

The SEMA format is a structured ZIP archive. Every layer has a purpose. Every purpose is permanent.

LAYER 01

Content

The original file — untouched, uncompressed. PDF, DOCX, JPG, MP4, anything. The soul of the archive.

content/ · Original format preserved
LAYER 02

Metadata

Author, creation date, language, tags, source, version, and semantic context. Rich, structured, machine-readable.

meta.json · application/vnd.sema
LAYER 03

Intelligence

Pre-computed understanding of the file's content. Summaries, key concepts, Q&A pairs, extracted entities — all offline.

brain.json · Local AI, no API needed
LAYER 04

Viewer

A self-contained HTML interface. Open the file in any browser anywhere in the world and it renders itself perfectly.

view.html · Zero dependency rendering
